1. Codex 是什么？一句话讲明白

Codex 可以理解为一个会看文件、会写代码、会整理报告的电脑助手。它不是“替你写完所有东西的人”，而是帮你把麻烦步骤做快一点。

你可以让它做：检查表格、清洗数据、生成图表、总结文献、改写段落、检查报告问题。
你不能完全交给它：最终结论、引用真假、数据是否合理、论文是否符合导师要求。

2. 安装：用网站 exe，不用命令行

小白版安装只走官网下载安装包。不要先学终端，不要复制命令，先把软件装起来。

打开官网 Quickstart 访问 https://developers.openai.com/codex/quickstart。
找到 Codex App 的下载入口 选择 Windows 版本，下载网页提供的安装程序。优先选择官网页面上的 Windows 安装包或下载按钮。
双击 exe 安装 下载完成后，打开安装包，按提示下一步安装。如果系统弹出安全确认，确认来源是官网后继续。
打开 Codex 并登录 用你的 ChatGPT/OpenAI 账号登录。第一次打开时，选择一个项目文件夹。
如果官网页面后来改版，没有直接显示 exe，就按官网页面提供的 Windows 下载方式操作。安装入口以 OpenAI 官方 Quickstart 为准。

5. 数据分析：让 Codex 像助教一样帮你

数据分析不要一上来就说“帮我分析”。要先让它看数据，再让它提建议，最后再写图表和结论。

看数据多少行多少列
找问题缺失和重复
清洗生成新表
画图看趋势
解释写报告
看懂 Excel
请帮我看这个 Excel 表格。告诉我：
1. 有多少行、多少列；
2. 每一列大概是什么意思；
3. 有没有空值、重复值或奇怪的值；
4. 如果我要做分析，下一步应该先做什么。

请先不要改文件。
画一张图
请根据这个数据，帮我推荐一张最适合小白理解的图。
先说明为什么选这张图，再生成图表。
图表标题、横轴、纵轴都要写清楚。

6. 文献阅读：让 Codex 帮你抓重点

读论文时，不要让 Codex 只“总结一下”。更好的方式是让它帮你找：这篇文章研究什么、用了什么数据、怎么分析、结论是什么、有什么局限。

论文阅读小白提示词
请用小白能懂的话帮我读这篇论文。

请回答：
1. 这篇论文想解决什么问题？
2. 它用了什么数据或材料？
3. 它用了什么研究方法？
4. 最重要的结论是什么？
5. 这篇论文有什么不足？
6. 哪些内容可以放进我的论文或报告？

不要编造原文没有的信息。

7. 论文撰写：先搭骨架，再写句子

小白写论文最容易卡住，是因为一上来就想写完整段落。更简单的方法是先让 Codex 搭一个“骨架”。

题目写什么
背景为什么重要
方法怎么做
结果发现什么
讨论说明什么
写作骨架提示词
我现在要写一份小论文/研究报告。
主题是：【写你的主题】。
我已有的数据/图表是：【简单说明】。
我读过的文献大概说：【简单说明】。

请先帮我搭一个小白能照着写的提纲。
不要直接写完整论文。
改写段落提示词
请把下面这段话改得更像论文语言，但不要增加新事实，不要夸大结论。

原文：
【粘贴你的段落】

8. 最重要的安全线：别让 AI 替你做判断

查

查引用

论文、作者、年份、数据都要回到原文确认。

看

看数据

结果好不好看不重要，数据是否真实合理更重要。

问

问自己

这句话有没有证据？这张图有没有误导？

最简单的检查方法：凡是 Codex 写出的结论，都问一句“这句话来自哪个数据、哪张图、哪篇论文？”
